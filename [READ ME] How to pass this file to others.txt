このファイルを他人へ渡す方法

  何故かはわからないが，歴代の先輩方はNASUの中にcppファイルとhファイルしか置いておらず，
  slnファイル(visual studioの構成ファイル)などがない.

  おそらく，visual studioの使用上作業フォルダ全体の容量が大きくなる(800MB～2GB以上)ためだと思うが，
  これらは配布に不必要なフォルダを削除することで10MB程度まで減らすことが可能である．

  参考 https://qiita.com/Narapi/items/f6b12e99e4cf8a3214b4 (アクセス日 2023/08/23)
  Referenceフォルダにもおいておく，ファイル名 Reducing the size of the project folder in Visual Studio C++

  ・x64 or x84 フォルダ
    おそらくDebugやRelese などが入っているはず．削除してOK
    ソリューションフォルダのなかにも存在していると思うので，これも削除してok
    ビルド/コンパイルしたオブジェクトファイルなどが入っているだけなのでビルドしなおせば復活する．

  ・.vs フォルダ(隠しファイル)
    重たくなる原因．平気で数GBぐらいある．
    削除してもOK．
    確か.vsフォルダには，visual studioの設定(どのファイルを開いていたか，とかどのコメントを折りたたんでいたか)や，
    バックアップのデータが入っていたはず．
    隠しフォルダ/ファイルの表示方法はググってクレメンス．win7 win10 win11ごとに違うので．


とはいえ，ファイルをそのまま渡すのはスマートではない，ﾀｲﾍﾝﾀﾞｼﾈ．
gitを使うのが良いと思うので私はgitを使おうと思う．(gitは使い方がわかりづらいので，忙しい設計研の人は手を出しにくいが...)
